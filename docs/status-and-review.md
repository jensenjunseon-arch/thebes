# Thebes AI — 총정리 & 점검 (2026-06-01)

이때까지의 작업을 한 번에 정리하고, 코드·사업·기술을 점검해 개선점을 도출한다.

---

## 1. 총정리 — 지금 제품은 무엇인가

### 핵심 피벗의 역사
1. **시작**: "영어로 수학을 *푸는*" 폴리아 4단계 튜터링 세션 (끝없는 채팅)
2. **재설계(3빌드)**: 바운디드 진단으로 — 온보딩 → 세션 → 결과 리빌
3. **결정적 피벗**: "문제를 풀게 하니 답이 애매" → **답을 구하지 않고 사고를 바깥으로 넓히는 아크**

### 현재 진단 플로우
```
[인트로]  AI 시대 프레이밍 + 6가지 사고력 staggered reveal
   ↓  예시 문제로 시작  /  📷 내 문제 사진 올리기
[세션]   2스텝 · 5단계 widening arc (계산 없음)
   재정의 → 구성요소 분해 → 관계 파악 → 실생활 연결 → 확장적 사고
   · 첫 질문 후킹 + 한국어 온램프 + 시작문장 칩
   · 매 턴: 증거카드 + 콤보 탤리(6칸) + 완성 게이지
[결과]   AI 인재 리포트 — 레이더 + 한 문장 프로필 + 본인 인용
   ↓  영어로 정리하고 따라쓰기 (상단 강조)
[리캡]   대화 전체 → 영어 문단 · 따라쓰기(실시간 일치율) · 내 문장 점검
         · 읽어주기(리스닝)/따라 말하기(스피킹) = '곧' 플레이스홀더
```

### 측정하는 6가지 사고력 (구인)
재정의 · 구성요소 분해 · 관계 파악 · 실생활 연결 · 확장적 사고 · 영어 추론 표현력
점수는 단계별 1~5점 graduated (언어 마커 + 길이 + 구체성).

### 기술 스택
Next.js 15(App Router) · TS · Tailwind v3 · Recharts · Supabase(Auth/PG, 미사용 대기) · Anthropic(라이브 코치, 크레딧 0으로 대기). **데모는 100% 클라이언트 스크립트 — 무API·무로그인·무전송.**

## 2. 코드 점검 — 발견된 이슈

### 🔴 P0 — 랜딩 ↔ 제품 메시지 불일치 (가장 시급)
랜딩(`app/page.tsx`, `app/layout.tsx`, `components/landing/PolyaDemo.tsx`)이 옛 모델을 말한다:
- "영어로 수학을 **풀게** 하는" / "폴리아의 **4단계**" / 옛 구인(풀이 경로 PATHS)
- 실제 진단은 "**풀지 않고** 사고를 넓히는 5단계 + 새 구인(구성요소·관계·실생활·확장)"
→ 방문자의 기대와 체험이 어긋남. **랜딩 카피·PolyaDemo·metadata를 새 서사로 재작성 필요.**

### 🟠 P1
- **DB enum 재마이그레이션 필요**: `construct_id` enum을 새 구인으로 바꿨지만, 이미 배포된 Supabase는 옛 값. 라이브(크레딧 필요) 켤 때 재적용해야 함. 데모는 무관.
- **라이브 AI 경로 divergence 위험**: `lib/ai/*`(tutor·scorer·prompts)는 새 모델로 업데이트했으나 크레딧 없어 미검증. 켜기 전 실제 호출 테스트 필요.
- **사진 object URL 미해제**: `URL.createObjectURL`만 쓰고 `revokeObjectURL` 없음 → 반복 업로드 시 경미한 메모리 누수.

### 🟡 P2 (정리/위생)
- `EvidenceByConstruct` 타입이 컴포넌트(`DiagnosticResult.tsx`)에 정의되어 `lib/recap.ts`가 역참조 — 타입을 `lib`로 옮기는 게 깔끔.
- `Signal`의 `"stuck"`이 "친절 칭찬 tier"로 재활용됨 — 의미 혼동, 이름 분리 권장.
- `ConstructRadar.tsx` 주석이 삭제된 ScorePanel 참조(무해).
- 자동화 테스트 0 — 엔진(분류/점수/플로우)은 node 시뮬로만 검증. 회귀 방지용 단위테스트 권장.
- 리캡 문단(커스텀 사진)이 "This problem is about the situation shown in your problem" 처럼 'problem' 중복 — GENERIC_COACHING.summary 다듬기.

### ✅ 양호
- `tsc --noEmit` 0 에러, 프로덕션 빌드 통과(9/9).
- 답 토큰 서버 전용 유지, leak 필터 숫자 경계 매칭.
- 데모 무API 동작 — 크레딧/백엔드 없이 끝까지 작동.

## 3. 사업성 점검

### 강점 (왜 설득력 있나)
- **타이밍**: "AI가 답을 내는 시대, 무엇을 기를까"는 학부모의 실제 불안을 정확히 친다.
- **차별화**: 경쟁(문제풀이 인강·학원)과 정면충돌 안 함 — "답이 아니라 사고·영어 표현"이라는 새 카테고리.
- **측정 가능한 무형 가치**: 사고력을 6구인 리포트로 가시화 → 학부모 결제 동인.
- **확장성 있는 후킹**: 내 문제 사진 업로드 → "콘텐츠 무한" 문제(문제은행 고갈)를 우회.

### 리스크 / 검증 필요
- **측정 타당도**: 현재 점수는 *언어 마커 기반 규칙*. "이게 진짜 사고력을 재는가?"는 미검증 — 투자자·교육자가 가장 먼저 찌를 지점. (라이브 LLM 채점 + 신뢰도/재현성 데이터 필요.)
- **WOW의 깊이**: 데모는 스크립트라 "내용을 정말 이해하는가?"가 약함. 실제 LLM 코치로 한 번은 보여줘야 진정성↑.
- **수익 모델 미구현**: 가입→리포트 저장 CTA만 있고 결제·구독 흐름 없음.
- **리텐션 루프**: "매주 쌓이면 리포트"가 핵심인데, 주간 재방문 동기/알림 없음.

### 결제 전환 가설
무료 진단(후킹) → AI 인재 리포트(가치 증명) → 가입(결과 저장) → 주간 구독(성장 추적). **지금은 1~2단계만 존재.** 3~4단계(저장·구독·주간 리포트)가 비어 있음.

## 4. 기술 점검

- **데모(스크립트) vs 라이브(LLM) 이원화**: 지금은 스크립트가 진실. 라이브는 빌드돼 있으나 미검증·divergence 위험. → 라이브를 "정답"으로 승격하려면 크레딧 + 프롬프트 실측 + DB 재마이그레이션.
- **번들**: `/session` 285kB, `/dashboard` 291kB first-load — Recharts가 무겁다. 진단엔 레이더 1개뿐이니 경량 SVG 레이더로 대체하면 ~150kB 절감 가능.
- **영속성 0**: 진단 결과가 어디에도 저장 안 됨(가입해도 결과 캡처 미구현). 리포트 사업의 핵심인데 비어 있음.
- **AV 기능(TTS/STT)**: 브라우저 내장 Web Speech API로 무료·무키 구현 가능 — '곧' 버튼을 실제로 켤 수 있음(크롬/사파리 지원).
- **OCR**: 사진→텍스트는 현재 불필요(사고 진단은 텍스트 무관). 다만 "문제를 읽고 맞춤 코칭"으로 가려면 Vision 필요.

## 5. 개선점 (우선순위)

### P0 — 지금
1. **랜딩 재정렬**: 새 서사("풀지 않는다, 사고를 넓힌다 / AI 인재 리포트")로 hero·PolyaDemo·metadata 재작성. 제품과 첫인상 일치.
2. **결과 영속화 + 가입 캡처**: 진단 결과를 저장하고 가입 시 첫 리포트로 연결 (수익 깔때기의 시작).

### P1 — 곧
3. **라이브 LLM 코치 1회 검증**: 크레딧 충전 후 실제 대화·채점이 스크립트만큼 자연스러운지 확인 → "진짜 AI" 데모 확보.
4. **TTS/STT 실연결**: Web Speech API로 읽어주기·따라말하기 활성화 (리스닝·스피킹 = 추가 가치 축).
5. **측정 타당도 근거**: 채점 루브릭 문서화 + (가능하면) LLM 채점의 신뢰도 데이터 — 투자자 방어.

### P2 — 다듬기
6. Recharts → 경량 레이더(번들 절감). 7. 엔진 단위테스트. 8. object URL revoke. 9. 타입 위치 정리. 10. 리캡 문단 카피 다듬기.

**후킹(무료 진단)과 가치 증명(AI 인재 리포트)까지는 강하게 만들어졌다. 다음 산은 ① 랜딩-제품 일치 ② 결과 영속화·수익 깔때기 ③ 측정 타당도(진짜 AI) — 즉 "데모"에서 "제품"으로 넘어가는 다리다.**
