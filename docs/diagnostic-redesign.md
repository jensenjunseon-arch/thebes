# 사고력 무료 진단 — 전체 재설계 플랜

> 대상: `/session/demo`
> 목표: "끝없는 튜터링 채팅"을 → "바운디드 진단 + 결과 리빌"로 전환
> 핵심: 와우포인트를 ① 시작 프레이밍 ② 실시간 증거 연결 ③ 결과 리빌 — 3곳에 심는다

---

## 1. 재정의 — 세션 → 진단

현재 `/session/demo`는 **끝이 없는 채팅**이다. 진단의 본질인 "빠르게 풀고 → 너에 대한 걸 보여주고 → 캡처 → 전환"이 없다.

진단(diagnostic)이 가져야 할 4가지 성질:

| 성질 | 현재 | 목표 |
|------|------|------|
| **바운디드** | 무한 채팅 | 4스텝 = 끝이 보임 |
| **프레이밍** | 없음 | 시작 시 "6가지 사고력을 본다" |
| **점진적 리빌** | 미세한 +3 막대 | 매 턴 "이 문장 → 이 사고 감지" |
| **페이오프** | 없음 | 레이더 + 한 문장 프로필 + 리포트 |

---

## 2. 전체 플로우 (3 phases)

```
[Phase 0] 온보딩          [Phase 1] 진단 세션         [Phase 2] 결과 리빌
 ~10초                     ~5분 / 4스텝                페이오프
┌───────────────┐         ┌───────────────┐          ┌───────────────┐
│ 5분 AI 진단    │   →     │ 코치 인사      │    →     │ 레이더 차트    │
│ 6가지 사고력   │         │ Frame/Plan/    │          │ 한 문장 프로필 │
│ 칩 미리보기    │         │ Work/Lookback  │          │ 강점·성장영역  │
│ [진단 시작]    │         │ 실시간 증거카드 │          │ 리포트 미리보기 │
└───────────────┘         │ 진행바 4/4     │          │ [가입하고 저장]│
                          └───────────────┘          └───────────────┘
```

상단 phase 상태 머신: `intro → session → result`

---

## 3. 와우포인트 3곳

### ① 온보딩 — "이건 진짜 측정 도구다"
6개 구인(재정의·가정·경로·검증·논리·영어표현)을 칩으로 먼저 보여준다.
→ 결과가 나왔을 때 "아, 그 6개를 본 거구나"로 납득됨. 측정의 정당성.

### ② 실시간 — "AI가 내 문장을 읽었다" (세션 중 핵심)
지금은 evidence 패널이 맨 아래 따로 떨어져 점수 막대와 연결이 안 보인다.
→ 점수가 움직일 때마다 카드가 슬라이드인:

```
┌─────────────────────────────────┐
│ 너의 문장                         │
│ "I'm assuming the speed is        │
│  constant the whole way."         │
│         ↓ 감지                    │
│ 🟠 가정 명시 (ASSUME)   +3        │
└─────────────────────────────────┘
```
문장 → 구인 → 점수의 인과를 *매 턴* 눈으로 본다. 제품의 진짜 마법.

### ③ 결과 — "이게 나구나" (마지막 페이오프)
- 레이더 차트가 애니메이션으로 그려짐
- **한 문장 프로필**: "넌 검증(VERIFY)에 강하고, 풀이 경로 탐색(PATHS)이 약해."
- 강점·성장영역 each + **세션에서 네가 실제로 한 말**을 근거로
- "한 문제로 본 스냅샷. 매주 쌓이면 →" 부모 리포트 미리보기(ReportPreview 재활용)
- CTA: "결과 저장하고 부모님 리포트 받기" → 가입 (결과 캡처)

## 4. 모바일 우선 원칙

코드에서 확인된 현재 문제 → 원칙:

| 현재 문제 | 원칙 |
|-----------|------|
| 영어 전용 라벨 (`Make the student restate…`) | UI·안내는 한글, 영어는 *콘텐츠*(문제·추론)에만 |
| `⌘/Ctrl+Enter`로만 전송 | 항상 보이는 전송 버튼 + Enter 전송, 모바일 힌트 제거 |
| `text-[10px]` 남발 | 본문 최소 14px, 안내문 10px 금지 |
| `min-h-[520px]` 고정 + 6타일 세로 스택 | 단일 컬럼, 채팅이 주인공 |
| 문제 카드가 옆에 큰 박스 | 상단 접이식 칩 (탭하면 펼침) |
| 점수 6타일이 흐름 중간 차지 | 슬림 라이브 바(6점), 풀 분해는 결과에서만 |
| 빈 영어 박스로 시작 | 코치 인사말 + 예시 칩 |

## 5. 화면별 설계

### Phase 0 — 온보딩

```
        사고력 진단

  5분이면, AI가 너의 수학 사고력을
  6가지로 진단합니다.

  [재정의] [가정] [경로]
  [검증] [논리] [영어표현]

  영어로 수학을 풀며, 점수가 아니라
  '어떻게 생각하는가'를 봅니다.

  ┌─────────────────────┐
  │   진단 시작하기  →    │
  └─────────────────────┘
  로그인 없이 바로 시작 · 약 5분
```

```
│ ●○○○  Step 1/4 · 문제 이해   │  ← 슬림 진행바
├─────────────────────────────┤
│ 코치: Let's start. What is   │
│ this problem actually asking?│  ← 인사말로 시작
│                              │
│              나: The problem │
│              wants the avg…  │
│                              │
│ ┌──────────────────────────┐│
│ │ "the avg speed of…"       ││  ← 실시간 증거카드
│ │  ↓ 재정의 감지   +4       ││

│ [영어로 생각을 적어보세요   ] │
```

### Phase 2 — 결과 리빌

```
        진단 완료

      너의 사고력 스냅샷

  ┌─────────────────────────┐
  │ 매주 이렇게 쌓이면 →      │
  │  (부모 리포트 미리보기)    │
```

### 재활용

### 신규 로직

- **결과 생성**: 세션 totals + 구인별 베스트 증거 → 한 문장 프로필 + 강점/성장. v0는 규칙 기반(최고/최저 구인), 추후 Claude 호출로 고도화
- **결과 캡처**: 가입 시 진단 결과를 첫 리포트로 저장

## 7. 빌드 순서 (설계는 전체, 구현은 단계)

| 빌드 | 내용 | 와우 |

## 8. 확정된 결정 (2026-05-30)

1. **진단 길이**: ✅ **2스텝 압축** — 단, 6개 구인을 다 측정하기 위해 *병합형*으로 설계
   - **Step 1 — 이해 & 계획**: 문제 재진술(redefine) + 가정 명시(assume) + 접근 1–2개(paths) + 영어(english)
   - **Step 2 — 풀이 & 검증**: 단계별 논리(logic) + 답 검증(verify) + 영어(english)
   - → 2스텝으로 6개 구인 전부 커버
2. **로그인 시점**: ✅ **무로그인 시작 → 결과 저장 시 가입**
3. **결과 생성**: ✅ **v0 규칙 기반** (최고·최저 구인 → 템플릿 한 문장). 추후 Claude 고도화
4. **데모 문제**: 현재 1개(평균속력) 유지

## 10. 대화 엔진 — 질문 흐름과 측정 방식 (the core)

`lib/diagnosticEngine.ts`. 무API, 클라이언트에서 실행. 실 AI로 전환해도 동일한 5단계 골격.

### 단계 → 질문 → 측정 구인

| Step | Stage | 코치 질문 (영어) | 측정 구인 |
|------|-------|------------------|-----------|
| 1 | restate | "what is this problem actually asking? Say it in your own words." | redefine |
| 1 | assume | "what are you assuming that the problem doesn't say out loud?" | assume |
| 1 | plan | "how would you get there? Is there more than one way?" | paths |
| 2 | reason | "walk me through your first step — and why that step?" | logic |
| 2 | verify | "you've reached an answer. how could you check it a different way?" | verify |
| 전 단계 | — | (영어 문장 품질) | english (보너스) |

restate·reason은 각 step의 greeting이 던지고, 코치 응답이 다음 stage 질문을 이어 붙임 → 한 메시지 = 한 질문, 끊김 없는 흐름.

### 측정 = 분류 (strong / partial / stuck)

학생 발화의 **언어적 마커**로 분류. 이게 곧 점수.
- **stuck**: "idk / 모르 / ? / 1단어 이하" → 점수 0, 그 문제에 맞는 **비계** 제시, 1회 더 기회
- **partial**: 마커는 있으나 짧음 → +2
- **strong**: 마커 + 충분한 길이(≥5–6단어) → +4
- **english 보너스**: 7단어 이상 완결 영어 문장 → +1 (헤드라인에선 제외)

마커 예: assume = `assume/suppose/if/same/constant/both…`, reason = `because/since/so/=/divide…`, verify = `check/units/makes sense/between…`.

### 비계 (stuck일 때, 문제별 — `Problem.coaching`)

- restate → `"${restateFrame}"` 채움형 프레임
- assume → `keyAssumption` 을 질문으로 ("그거 당연하게 여기고 있지 않아?")
- plan → `approaches[0]` vs `approaches[1]` 두 경로 제시 → 고르게
- verify → `verifyHint` (답은 절대 노출 안 함)

→ 비계가 "그 문제를 깊이 아는 코치"처럼 느껴짐. 답 숫자는 클라이언트에 안 나감(서버 전용 `forbiddenAnswerTokens`), 비계는 답-free라 안전.

### 진입장벽 ↓ + 흥미 유지 (와우 포인트)

- **시작 문장 칩**("I'm assuming that…", "One way is to…") → 빈 영어창 공포 제거 + 영어 추론 패턴을 *가르치며* 측정
- **실시간 증거 카드**: 학생이 친 실제 문장 → 감지된 구인 → +점수
- **막혀도 끝까지 간다**: stuck 2회면 자동 진행 → 데모가 절대 멈추지 않음
- **결과 헤드라인은 추론 구인**(증거 인용 보장) + balanced 분기

### 문제 풀

18문제 (초등 저/고학년 · 중1/2/3 · 고1, 각 3문제), 전부 `coaching` 메타데이터 보유. 난이도 칩 + "다른 문제 ↻"로 전환.

## 11. 피벗 (2026-06-01) — "문제 풀이"에서 "사고 확장 아크"로

수학 문제처럼 답을 강요하니 처음 온 사람이 답하기 애매했다. 진단을 **답을 구하지 않는, 사고를 바깥으로 넓혀가는 아크**로 전환:

재정의 → **구성 요소 분해** → **요소 간 관계** → **실생활 연결(나에게의 이득)** → **확장적 사고(주변·미래)**

이에 맞춰 6가지 측정 구인 자체를 교체:
- redefine 문제 재정의력
- **decompose 구성 요소 분해** (was assume)
- **relate 관계 파악** (was paths)
- **relevance 실생활 연결** (was verify)
- **transfer 확장적 사고** (was logic)
- english 영어 추론 표현력

2스텝: ① 이 상황 이해하기(재정의·구성요소·관계) ② 나와 세상에 연결하기(이득·확장).
숫자 답을 절대 묻지 않으므로 누구나 답할 수 있고, "AI가 못 해주는 너만의 이해와 연결"이라 사업 본질을 정확히 증명한다.

변경 파일: constructs.ts, steps.ts, diagnosticEngine.ts(5단계 재작성), problems.ts(32문제 coaching을 summary/components/relationship/realWorld로 교체), DiagnosticResult(GROWTH_TIP), scorer+prompts, ReportPreview(랜딩 일관성), mockData/queries/migration enum.
