# 사고력 무료 진단 — 전체 재설계 플랜

> 대상: `/session/demo`
> 목표: "끝없는 튜터링 채팅"을 → "바운디드 진단 + 결과 리빌"로 전환
> 핵심: 와우포인트를 ① 시작 프레이밍 ② 실시간 증거 연결 ③ 결과 리빌 — 3곳에 심는다

---

## 1. 재정의 — 세션 → 진단

현재 `/session/demo`는 **끝이 없는 채팅**이다. 진단의 본질인 "빠르게 풀고 → 너에 대한 걸 보여주고 → 캡처 → 전환"이 없다.

진단(diagnostic)이 가져야 할 4가지 성질:

| 성질 | 현재 | 목표 |
|------|------|------|
| **바운디드** | 무한 채팅 | 4스텝 = 끝이 보임 |
| **프레이밍** | 없음 | 시작 시 "6가지 사고력을 본다" |
| **점진적 리빌** | 미세한 +3 막대 | 매 턴 "이 문장 → 이 사고 감지" |
| **페이오프** | 없음 | 레이더 + 한 문장 프로필 + 리포트 |

---

## 2. 전체 플로우 (3 phases)

```
[Phase 0] 온보딩          [Phase 1] 진단 세션         [Phase 2] 결과 리빌
 ~10초                     ~5분 / 4스텝                페이오프
┌───────────────┐         ┌───────────────┐          ┌───────────────┐
│ 5분 AI 진단    │   →     │ 코치 인사      │    →     │ 레이더 차트    │
│ 6가지 사고력   │         │ Frame/Plan/    │          │ 한 문장 프로필 │
│ 칩 미리보기    │         │ Work/Lookback  │          │ 강점·성장영역  │
│ [진단 시작]    │         │ 실시간 증거카드 │          │ 리포트 미리보기 │
└───────────────┘         │ 진행바 4/4     │          │ [가입하고 저장]│
                          └───────────────┘          └───────────────┘
```

상단 phase 상태 머신: `intro → session → result`

---

## 3. 와우포인트 3곳

### ① 온보딩 — "이건 진짜 측정 도구다"
6개 구인(재정의·가정·경로·검증·논리·영어표현)을 칩으로 먼저 보여준다.
→ 결과가 나왔을 때 "아, 그 6개를 본 거구나"로 납득됨. 측정의 정당성.

### ② 실시간 — "AI가 내 문장을 읽었다" (세션 중 핵심)
지금은 evidence 패널이 맨 아래 따로 떨어져 점수 막대와 연결이 안 보인다.
→ 점수가 움직일 때마다 카드가 슬라이드인:

```
┌─────────────────────────────────┐
│ 너의 문장                         │
│ "I'm assuming the speed is        │
│  constant the whole way."         │
│         ↓ 감지                    │
│ 🟠 가정 명시 (ASSUME)   +3        │
└─────────────────────────────────┘
```
문장 → 구인 → 점수의 인과를 *매 턴* 눈으로 본다. 제품의 진짜 마법.

### ③ 결과 — "이게 나구나" (마지막 페이오프)
- 레이더 차트가 애니메이션으로 그려짐
- **한 문장 프로필**: "넌 검증(VERIFY)에 강하고, 풀이 경로 탐색(PATHS)이 약해."
- 강점·성장영역 each + **세션에서 네가 실제로 한 말**을 근거로
- "한 문제로 본 스냅샷. 매주 쌓이면 →" 부모 리포트 미리보기(ReportPreview 재활용)
- CTA: "결과 저장하고 부모님 리포트 받기" → 가입 (결과 캡처)

## 4. 모바일 우선 원칙

코드에서 확인된 현재 문제 → 원칙:

| 현재 문제 | 원칙 |
|-----------|------|
| 영어 전용 라벨 (`Make the student restate…`) | UI·안내는 한글, 영어는 *콘텐츠*(문제·추론)에만 |
| `⌘/Ctrl+Enter`로만 전송 | 항상 보이는 전송 버튼 + Enter 전송, 모바일 힌트 제거 |
| `text-[10px]` 남발 | 본문 최소 14px, 안내문 10px 금지 |
| `min-h-[520px]` 고정 + 6타일 세로 스택 | 단일 컬럼, 채팅이 주인공 |
| 문제 카드가 옆에 큰 박스 | 상단 접이식 칩 (탭하면 펼침) |
| 점수 6타일이 흐름 중간 차지 | 슬림 라이브 바(6점), 풀 분해는 결과에서만 |
| 빈 영어 박스로 시작 | 코치 인사말 + 예시 칩 |

## 5. 화면별 설계

### Phase 0 — 온보딩

```
        사고력 진단

  5분이면, AI가 너의 수학 사고력을
  6가지로 진단합니다.

  [재정의] [가정] [경로]
  [검증] [논리] [영어표현]

  영어로 수학을 풀며, 점수가 아니라
  '어떻게 생각하는가'를 봅니다.

  ┌─────────────────────┐
  │   진단 시작하기  →    │
  └─────────────────────┘
  로그인 없이 바로 시작 · 약 5분
```

### Phase 1 — 진단 세션 (모바일)

```
┌─ Session · 진단 ────────────┐
│ ▸ 문제 보기 (탭하면 펼침)     │  ← 접이식 칩
├─────────────────────────────┤
│ ●○○○  Step 1/4 · 문제 이해   │  ← 슬림 진행바
├─────────────────────────────┤
│ 코치: Let's start. What is   │
│ this problem actually asking?│  ← 인사말로 시작
│                              │
│              나: The problem │
│              wants the avg…  │
│                              │
│ ┌──────────────────────────┐│
│ │ "the avg speed of…"       ││  ← 실시간 증거카드
│ │  ↓ 재정의 감지   +4       ││
│ └──────────────────────────┘│
├─────────────────────────────┤
│ [영어로 생각을 적어보세요   ] │
│ [                    전송 ] │
└─────────────────────────────┘
   사고력  ▱▱▱▰▰  (탭하면 펼침)   ← 하단 슬림 점수바
```

### Phase 2 — 결과 리빌

```
        진단 완료

      너의 사고력 스냅샷
   ┌───────────────────┐
   │     (레이더 차트)   │  ← 애니메이션
   │   6개 구인 다각형    │
   └───────────────────┘

  "넌 검증에 강하고,
   풀이 경로 탐색이 약해."

  ── 강점 · 검증 ──────────
  "Let me check: 48×5=240 ✓"
   답을 다른 방법으로 확인함

  ── 성장 · 풀이 경로 ──────
  한 가지 방법에만 머물렀어요.
  다음엔 2가지 경로를 떠올려봐요.

  ┌─────────────────────────┐
  │ 매주 이렇게 쌓이면 →      │
  │  (부모 리포트 미리보기)    │
  └─────────────────────────┘

  ┌─────────────────────────┐
  │ 결과 저장 · 부모님 리포트 │ → 가입
  └─────────────────────────┘
```

## 6. 기술 구조

### 새 파일 구조
```
components/diagnostic/
  DiagnosticFlow.tsx      # phase 상태머신 (intro→session→result)
  DiagnosticIntro.tsx     # Phase 0
  DiagnosticSession.tsx   # Phase 1 (SessionView 모바일 리팩터)
  DiagnosticResult.tsx    # Phase 2 (리빌)
  LiveEvidenceCard.tsx    # 문장→구인→점수 카드
  LiveScoreBar.tsx        # 슬림 하단 점수바
  ProblemChip.tsx         # 접이식 문제 칩
```

### 재활용
- `lib/constructs.ts`, `lib/steps.ts` — 그대로
- `submitTurn` 액션 + 스코어러 — 그대로
- `ReportPreview` (랜딩) — 결과의 부모 리포트 티저로
- Recharts 레이더 — 결과 차트로

### 신규 로직
- **코치 인사말**: 각 스텝 진입 시 코치 오프닝 턴 seed
- **결과 생성**: 세션 totals + 구인별 베스트 증거 → 한 문장 프로필 + 강점/성장. v0는 규칙 기반(최고/최저 구인), 추후 Claude 호출로 고도화
- **결과 캡처**: 가입 시 진단 결과를 첫 리포트로 저장

## 7. 빌드 순서 (설계는 전체, 구현은 단계)

| 빌드 | 내용 | 와우 |
|------|------|------|
| **1** | 모바일 우선 세션 리팩터 + 코치 인사말 + 실시간 증거카드 | ② 실시간 |
| **2** | 결과 리빌 + 결과 생성 + 가입 캡처 | ③ 결과 |
| **3** | 온보딩 인트로 + 애니메이션 폴리시 | ① 프레이밍 |

빌드 1이 "지금 있는 걸 모바일에서 쓸만하게 + 실시간 마법 추가",
빌드 2가 "진짜 페이오프", 빌드 3이 "마감".

## 8. 확정된 결정 (2026-05-30)

1. **진단 길이**: ✅ **2스텝 압축** — 단, 6개 구인을 다 측정하기 위해 *병합형*으로 설계
   - **Step 1 — 이해 & 계획**: 문제 재진술(redefine) + 가정 명시(assume) + 접근 1–2개(paths) + 영어(english)
   - **Step 2 — 풀이 & 검증**: 단계별 논리(logic) + 답 검증(verify) + 영어(english)
   - → 2스텝으로 6개 구인 전부 커버
2. **로그인 시점**: ✅ **무로그인 시작 → 결과 저장 시 가입**
3. **결과 생성**: ✅ **v0 규칙 기반** (최고·최저 구인 → 템플릿 한 문장). 추후 Claude 고도화
4. **데모 문제**: 현재 1개(평균속력) 유지

## 9. 2스텝 구인 매핑

| Step | 이름 | 의도 | primary constructs |
|------|------|------|--------------------|
| 1 | 이해 & 계획 (Understand) | 재진술·가정·접근 | redefine, assume, paths, english |
| 2 | 풀이 & 검증 (Solve) | 단계 논리·검증 | logic, verify, english |
